# CLAUDE.md

Conventions and guardrails for the Korea Real-Estate Investment-Firm OS
monorepo. Read this once before making any changes — most "weird" decisions
in this codebase are documented here, and reproducing them keeps the
system consistent.

## Layout

- `apps/web` — the only active product. Next.js 15 App Router, Prisma 5,
  Postgres 16. All new product code lives here.
- `packages/contracts` — Hardhat workspace for the on-chain registry and
  ERC-3643-style tokenization stack. Web reads ABIs auto-exported from
  this package (see "ABI" section below).
- `legacy/` — archived previous demos. Do not import from here.
- `docs/`, `apps/web/docs/` — operating runbooks and architecture notes.
  `apps/web/docs/production-runbook.md` is the source of truth for prod
  hardening; `apps/web/DEPLOYMENT.md` for the Vercel scaffold.

## Database (Prisma)

- The migration chain is currently the source of truth. `npm --prefix
  apps/web run typecheck && npx --prefix apps/web prisma migrate deploy`
  must succeed against an empty database.
- **Never** edit a migration that has already shipped. Reconcile drift via
  a new migration with `IF NOT EXISTS` / `IF EXISTS` guards. The
  `20260428080000_reconcile_schema_drift` migration is the model.
- After editing `schema.prisma`, run `npx prisma migrate dev` to generate
  the matching migration. The CI job `web-ci.static-checks` enforces
  schema-migration parity via `prisma migrate diff --exit-code`.
- All `model X` definitions must have a corresponding `CREATE TABLE`
  somewhere in `prisma/migrations/`.

## Auth

- Browser admins authenticate via signed session cookies
  (`ADMIN_SESSION_SECRET`). The `middleware.ts` file is the single gate.
- Public routes are listed in `isPublicApiPath` / `isPublicAdminPath`. Add
  to those lists rather than punching holes in the auth check inside the
  handler.
- New admin API routes should use `withAdminApi` (in
  `lib/security/with-admin-api.ts`) rather than re-implementing auth +
  audit + zod boilerplate.
- `ADMIN_ALLOW_UNBOUND_BROWSER_SESSION` is a dev-only escape hatch and
  must be unset in production. The production preflight rejects it.

## Edge protection

- IP allowlist + per-IP rate limit live in
  `lib/security/edge-protection.ts` and are wired in `middleware.ts`.
- Limits are configurable via `ADMIN_IP_ALLOWLIST`, `OPS_IP_ALLOWLIST`,
  and the `*_RATE_*` env pair. Defaults are conservative single-tenant
  values; tighten further behind a real WAF.
- For Node-runtime handlers needing distributed rate-limiting, use
  `checkDistributedRateLimit` from `lib/security/distributed-rate-limit.ts`
  (Upstash Redis REST). Soft-fails open when not configured.

## Storage

- `lib/storage/local.ts` exposes `createLocalDocumentStorage`,
  `createS3DocumentStorage`, and `createDocumentStorageFromEnv`. The
  env-aware factory auto-selects based on `DOCUMENT_STORAGE_BUCKET` and
  hard-blocks local FS in production.
- New code must call `createDocumentStorageFromEnv()` rather than the
  local-FS factory directly.
- Stored paths in DB are opaque: `s3://bucket/key` for S3, cwd-relative
  for local. Use `storage.read(path)` / `storage.presignedUrl(path)` to
  resolve at read time.

## Blockchain

- Hand-curated ABI in `apps/web/lib/blockchain/abi.ts` must match the
  artifact `abi.json` generated by `npm run contracts:export-abi`. CI
  runs `scripts/check-registry-abi.ts` to enforce parity.
- Tokenization writes go through `isTokenizationMockMode()` in
  `lib/blockchain/mock-mode.ts`. Mock mode is hard-blocked in production
  by both `getBlockchainConfig` and `isBlockchainMockMode`.
- Mock txHashes are deterministic (`buildMockTxHash(...)`) so audit logs
  remain reproducible across replays.

## Logging & observability

- Use `logger` from `lib/observability/logger.ts`. It emits JSON lines
  compatible with Vercel Log Drains, Datadog, Logtail, etc.
- Wrap request-scoped work with `withRequestContext({ requestId }, fn)`
  so every nested log line gets the trace id automatically. The
  middleware sets `X-Request-Id` on every response.
- `reportError(error, context)` forwards to the optional
  `ERROR_REPORT_WEBHOOK_URL` and never throws.
- Ops alert dedup: callers can use `computeOpsAlertFingerprint(payload)`
  + `isDuplicateOpsAlert(fingerprint)` to suppress duplicates within
  `OPS_ALERT_DEDUP_WINDOW_MINUTES` (default 30).

## Testing

- Unit tests are `tsx --test`. Run via `npm test`. They must remain
  network- and DB-free (use Prisma fakes).
- E2E is Playwright. `npm run e2e` boots Postgres + Next dev server.
- Time-sensitive tests must use `new Date()` or fixed offsets relative
  to it, never hard-coded calendar dates — the latter decay.

## Production gates

- `npm run prod:preflight` (run with prod env loaded) must pass before
  promoting a deployment to production traffic.
- The complete operational checklist lives in
  `apps/web/docs/production-runbook.md`.

## What to do BEFORE editing

1. `npm --prefix apps/web run prisma:generate`
2. `npm --prefix apps/web run typecheck`
3. `npm --prefix apps/web test`
4. Make the change.
5. Re-run 2–3.
6. `npm --prefix apps/web run build` for any change that crosses
   route boundaries or changes the middleware.
