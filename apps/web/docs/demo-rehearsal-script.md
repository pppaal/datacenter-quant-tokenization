# 데모 리허설 대본 (심사장용, 클릭+멘트)

> 목적: 창업지원 심사 발표에서 **5~7분 라이브 데모**를 무실수로. 클릭 순서·화면·멘트·대비책.
> 전체 기능 투어는 `demo-script.md`. 이 문서는 **심사장 압축 버전 + 실패 대비**입니다.
>
> 리허설 원칙: 발표 직전 `cd apps/web && npm run dev` 로 아래 4스텝을 **실제로 1회씩** 클릭.
> seed가 비었으면 `npm run prisma:seed`. 네트워크 의존 줄이려면 키 없이도 도는 mock 유지.

## 사전 준비 (발표 30분 전)

- [ ] `npm run dev` 띄우고 `/`, `/property-analyze`, `/admin/login` 셋 다 열어 캐시 워밍
- [ ] 운영자 로그인 1회 미리 (세션 쿠키) → 데모 중 로그인 화면에서 막히지 않게
- [ ] 브라우저 탭 4개 미리 열어두기 (아래 스텝별 1개) — 라이브 타이핑 최소화
- [ ] 화면 확대(125~150%), 폰트 크게, 시크릿창 알림/확장프로그램 끄기
- [ ] 인터넷 끊겨도 되도록 mock 경로로 (실데이터 키는 데모에서 굳이 안 켜도 됨)

## 핵심 메시지 (데모 내내 1줄로 관통)

> "PPT가 아니라 **작동하는 기관급 부동산 운용 OS**입니다. 리서치부터 토큰화 레지스트리까지
> 한 시스템에서 돕니다. 지금은 시연용 데이터로 돌고, **실데이터·체인·KYC 연동이 이번 지원금
> 용도**입니다."

---

## STEP 1 — AI 언더라이팅 (가장 임팩트 큼) · ~2분

**화면:** `/property-analyze`
**클릭:**

1. 주소 입력란에 데모 주소 입력 (또는 좌표) → 분석 실행
2. 결과 리포트 스크롤: Headline → 현금흐름(IS/CF) → 부채 스케줄/약정 → **DCF·IRR** → **GP/LP 워터폴** → 데이터 품질 패널
   **멘트:**

- "주소 하나로 지오코딩 → 다중 데이터 커넥터 → DCF·몬테카를로·워터폴까지 자동 산출됩니다."
- "사람이 엑셀로 며칠 하던 기관급 언더라이팅을 수 초 만에. 숫자는 시나리오·민감도까지 들어갑니다."
  **대비책:** 분석이 느리면 미리 한 번 실행해 **캐시 히트** 상태로 보여주기 (x-cache: hit).

## STEP 2 — 운용사 워크플로우 (딜→IC) · ~2분

**화면:** `/admin/deals` → 딜 1건 → `/admin/ic`
**클릭:**

1. `/admin/deals`에서 seed 딜 진입 → 단계(stage)·언더라이팅·체크리스트
2. `/admin/ic`에서 IC 패킷 → 의사결정/잠금/릴리즈 흐름, 감사추적
   **멘트:**

- "리서치·증빙리뷰·언더라이팅·**투자심의(IC) 거버넌스**·딜 실행이 한 시스템에. 모든 변경은
  해시체인 감사추적으로 남습니다 — 기관·LP 신뢰의 핵심."

## STEP 3 — 기관급 산출물: 재무제표 Excel · ~1분

**화면:** `/admin/assets/[id]` (또는 재무제표 익스포트 버튼)
**클릭:** 재무제표 Excel 다운로드 → **실제 .xlsx 파일이 열리는 것**을 보여주기
**멘트:**

- "PDF가 아니라 **진짜 엑셀**입니다. 숫자가 숫자로 들어가서 LP·IC가 바로 피벗·재계산.
  카운터파티별로 시트가 분리돼 비교 가능합니다."

## STEP 4 — 토큰화 차별점: 온체인 레지스트리 · ~1분

**화면:** `/admin/registry`
**클릭:** RWA 프로젝트/온체인 레코드 → 문서 해시 앵커링 / NAV 어테스테이션 상태
**멘트:**

- "문서·밸류에이션을 오프체인에 두고 **해시만 온체인 앵커링**(registry-only, ERC-3643 스택).
  소매 토큰판매가 아니라 **기관 RWA 인프라**입니다."
- (질문 받으면) "Waterfall 컨트랙트는 외부 감사 후 활성화 — 그게 지원금 사용처 중 하나."

## 예상 질문 & 정직한 답

- **"데이터 진짜 있어요?"** → "거시·하자드·POI는 키 없이 라이브 가능. 실거래/공시 등은
  발급키만 꽂으면 됨(`DATA_KEYS.md`). 지금 데모는 결정적 mock으로 end-to-end 검증된 상태."
- **"실제로 토큰 발행돼요?"** → "엔진·레지스트리 완성, mock-mode로 흐름 검증 완료. 실 체인
  배포 + Waterfall 외부감사 + KYC 연동이 남았고 = 지원금 용도."
- **"매출/고객은?"** → [채우기: 파일럿 LOI·인터뷰·파이프라인]. 정직하게 현 단계 말하기.
- **"경쟁사 대비?"** → "PPT 단계가 아니라 **작동하는 풀스택**(데이터→언더라이팅→딜→토큰화)이
  이미 통합돼 있다는 점."

## 절대 금지

- mock/seed를 "라이브 실데이터"라 말하기 → 찌르면 신뢰 붕괴.
- 안 켜진 기능을 켜진 척. 모르는 수치 즉답 → "확인 후 회신"이 안전.
