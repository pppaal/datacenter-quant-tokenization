# 한국 RWA 부동산 서비스 블루프린트 (실행형)

## 1. 실제 서비스 구조도

```text
[고객 채널]
  ├─ 랜딩/자산 리스트/자산 상세
  ├─ 상담 신청/리드 폼
  └─ 투자자 온보딩(KYC 연동 예정)
          │
          ▼
[애플리케이션 레이어]
  ├─ 자산 카탈로그 API
  ├─ 투자 메모 AI 생성 API
  ├─ 데이터룸 문서 API
  ├─ 리드 저장 API
  └─ 관리자 대시보드 API
          │
          ▼
[데이터 레이어]
  ├─ assets(자산/수익/리스크)
  ├─ dataroom(실사/임대차/평가 문서)
  └─ leads(문의/소스/상태)
```

---

## 2. 고객

- 개인 투자자: 소액 분산투자, 정기 캐시플로우 선호
- 고액/법인 투자자: 포트폴리오 다변화, 대체투자 접근
- 자산 보유자(오너): 자산 유동화/부분 엑시트 니즈
- 파트너: 법무법인, 회계법인, 자산관리사, 신탁/수탁 기관

---

## 3. 수익모델

1. 자산 소싱/구조화 수수료
2. 발행/플랫폼 이용 수수료
3. 운용관리 수수료(월/분기)
4. 성과 기반 수수료(사전 고지형)
5. B2B 화이트라벨/API 제공 수수료

---

## 4. 법률/규제 구조

- 자본시장법 이슈: 증권성 판단, 공시/설명의무
- 전자증권/수익증권 구조 검토: 권리 귀속 및 이전 방식 명확화
- AML/KYC: 고객확인, 거래모니터링, 의심거래 대응 체계
- 개인정보보호: 최소수집·보관기간·파기정책
- 광고/마케팅 컴플라이언스: 과장수익 표현 금지, 리스크 고지 의무

> 실서비스 전 변호사/회계사/컴플라이언스 책임자 검토를 필수 전제로 한다.

---

## 5. 운영 플로우

1. **자산 발굴/예비심사**
2. **실사(법무/세무/기술) + 밸류에이션**
3. **상품 구조화 + 투자메모 작성**
4. **데이터룸 공개 + 투자자 문의 대응**
5. **청약/배정 + 사후 리포팅**
6. **배당/성과 공시 + 재투자 유도**

---

## 6. MVP 기능 상세설계

### 6.1 자산 등록
- 필드: 자산명, 위치, 유형, 목표수익률, 최소투자금, 핵심 리스크, 상태
- 권한: 관리자만 등록/수정
- 출력: 리스트/상세 페이지 자동 반영

### 6.2 투자 메모 AI 생성
- 입력: 자산 정보(입지, 수익률, 공실률, 임차인 안정성, 리스크)
- 처리: 규칙 기반 템플릿 + 리스크 강조 요약
- 출력: “투자 포인트/주의 포인트/적합 투자자 유형” 문서

### 6.3 데이터룸
- 항목: 자산개요서, 임대차 요약, 감정평가서 요약, 리스크 체크리스트
- 정책: 공개문서/요청문서 구분, 접근권한 로그 추적(차기)

### 6.4 문의/리드 관리
- 폼: 이름, 이메일, 관심자산, 문의내용, 유입경로
- 저장: 서버 파일(JSON) -> 향후 CRM 연동
- 상태: 신규/응대중/전환/종결

### 6.5 관리자 페이지
- KPI: 총 리드, 오늘 리드, 자산별 관심도, 최근 문의
- 기능: 리드 목록 조회, 우선순위 응대 대상 식별

## 7. 내 추천은 (우선순위)

1. **지금 즉시**: 자산 3개 기준으로 리스트/상세/문의 백엔드 동작까지 완성
2. **2주 내**: 관리자 리드 상태변경 + 메모 기능 추가
3. **4주 내**: KYC/전자서명/알림(메일·문자) 연동
4. **6~8주 내**: 투자자 리포팅 자동생성 + 배당 시뮬레이터 배포

## 8. 블록체인 기술은 뭐가 좋을까? (현실적인 추천)

### 결론(추천)
- **초기 MVP/상용화 1순위: 이더리움 호환 EVM 계열 + L2(Polygon PoS, Arbitrum, Base 등)**
- 이유: 개발자 풀이 크고, 지갑/커스터디/감사 도구가 성숙했으며, 토큰 표준(ERC-20/1400 계열)과 연동 생태계가 가장 풍부함.

### 후보별 장단점
1. **Ethereum Mainnet**
   - 장점: 보안·신뢰도 최고, 글로벌 표준
   - 단점: 가스비 부담이 커 소액·고빈도 트랜잭션에 비효율
2. **EVM L2 (Polygon/Arbitrum/Base)**
   - 장점: 낮은 수수료, 빠른 처리, 이더리움 도구 재사용 가능
   - 단점: 체인별 브리지/운영 정책 차이 관리 필요
3. **Permissioned 체인(예: Hyperledger Besu/Quorum)**
   - 장점: 참여자 통제·프라이버시·컴플라이언스 대응이 쉬움
   - 단점: 퍼블릭 유동성/개방형 생태계 연결이 약함
4. **국내/기타 퍼블릭 체인**
   - 장점: 특정 파트너십/수수료 이점이 있을 수 있음
   - 단점: 장기 생태계 안정성·도구 성숙도 검증 필요

### 한국 RWA 부동산 기준 의사결정 프레임
- 우선순위 1: **규제/감사 대응** (권한통제, 로그추적, 동결/해제 정책)
- 우선순위 2: **운영 안정성** (멀티시그, 키관리, 모니터링)
- 우선순위 3: **총소유비용(TCO)** (가스비 + 감사비 + 운영비)
- 우선순위 4: **확장성** (2차 유통, 기관 API, 해외 투자자 접근)

### 실무적으로 가장 안전한 시작안
- 1단계: **오프체인 권리관리 + 온체인 증빙(해시/이력)** 하이브리드
- 2단계: EVM L2에서 발행/락업/배당 자동화
- 3단계: 규제 정합성 확인 후 유통 기능 점진 오픈

> 한 줄 추천: “지금은 EVM L2로 시작하고, 규제·감사 요건은 오프체인 통제로 보완하는 하이브리드 모델이 가장 현실적.”


## 9. 단계별 구현 상태 (요청 반영)

### 2단계: 자산 데이터 정리
- `name, assetType, country, city, address, status, powerCapacityMW, landArea, grossFloorArea, tenantStatus, capex, opex, expectedIRR, targetEquity, debtStructure, summary, riskNotes` 필드 저장.
- 목표: 딜 구조화 가능성 검증을 위한 정량/정성 정보 동시 제공.

### 3단계: AI 투자 메모 자동 생성
- 입력: 자산 정보 + 전력/입지/임차인/비용 데이터 + 리스크 항목.
- 출력: 한 줄 요약, 투자 포인트 3개, 주요 리스크 3개, DD 체크항목, 투자자용 메모.
- 목표: 기존 수작업 검토를 빠르게 압축.

### 4단계: 데이터룸
- PDF 업로드(데모), IM/DD/계약서/전력문서 카테고리화, 자산별 문서 묶음, 권한(public/investor/admin) 분리 조회.
- 목표: 실제 딜 플랫폼과 유사한 운영 경험 제공.
